My 3-Column Grid Nightmare

After Station 3, I felt unstoppable! I had intrinsic widths, content-based sizing, AND fluid spacing. Time to build a real project: a portfolio card grid. I wanted 3 columns, so I wrote:

My "perfect" 3-column grid:
.grid {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
  gap: var(--space-md);
}

/* Translation: "Always 3 equal columns" */

On my 1440px laptop: Beautiful! Three equal columns, perfectly spaced. I shipped it to the client.

HAP's 3-Column Disaster:

Client response: "HAP, the grid breaks on my phone and looks empty on my ultrawide monitor."

Mobile (375px): 3 columns = 125px each (minus gaps). Cards were squished, text overflowed, images crushed.

Ultrawide (3440px): 3 columns = 1146px each. Cards swam in whitespace. Huge gaps between items.

I tried fixing with media queries: 1 column for mobile, 2 for tablet, 3 for desktop, 4 for ultrawide... I wrote TWENTY LINES of media queries just to handle different screen sizes!

Prof. Teeters' Intervention

"HAP, stop. You're TELLING the browser 'always 3 columns.' What if the screen is too narrow for 3? Too wide for only 3? Don't tell it HOW MANY columns. Tell it the RULES for sizing, and let IT calculate how many fit."

She showed me one line:

.grid {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(16rem, 1fr));
  gap: var(--space-md);
}

Me: "Wait... I don't say '3 columns'? I say 'as many as fit with 16rem minimum'?!"

Prof. Teeters: "Exactly! Mobile (375px)? Browser calculates: 1 column fits. Tablet (768px)? 2 columns fit. Desktop (1440px)? 5 columns fit. Ultrawide (3440px)? 10+ columns fit. The browser is smarter than you at math!"

The Magic Pattern Explained

Prof. Teeters sat me down and broke apart the magic pattern piece by piece:

The Magic Pattern:
.grid {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(16rem, 1fr));
  gap: var(--space-md);
}

Part 1: repeat(...)

What it does: Repeats a track pattern

Old way: repeat(3, 1fr) - repeat pattern 3 times

New way: repeat(auto-fit, ...) - repeat pattern as many times as fit!

Part 2: auto-fit

What it does: Create as many tracks as fit in container

How it works:

  • Container width: 1440px
  • Minimum track size: 16rem (256px)
  • Gap: 1.5rem (24px)
  • Calculation: (1440 - gaps) √∑ 256 = ~5 columns

Browser does this math automatically for ANY container width!

Part 3: minmax(16rem, 1fr)

What it does: Each track minimum 16rem, maximum equal share (1fr)

Minimum (16rem): "Never smaller than 16rem, even if it means wrapping to fewer columns"

Maximum (1fr): "Grow to equal share of leftover space"

Result: Columns are always at least 16rem, but grow equally to fill space!

Part 4: The Result

What happens: Grid generates perfect column count for ANY width

  • 320px viewport: 1 column (320px < 512px for 2 columns)
  • 768px viewport: 2-3 columns (depending on gaps)
  • 1440px viewport: 5 columns
  • 3440px viewport: 12+ columns

No media queries! Just smart rules!

auto-fit vs auto-fill (The Difference):

Prof. Teeters showed me TWO keywords:

auto-fit:

Collapses empty tracks, expands filled tracks

Use when: Items should fill available space (most common)

/* 3 items in 1920px viewport */
repeat(auto-fit, minmax(16rem, 1fr))
/* Result: 3 wide columns (each ~640px) */
auto-fill:

Keeps empty tracks, maintains column structure

Use when: Consistent column count needed even with few items

/* 3 items in 1920px viewport */
repeat(auto-fill, minmax(16rem, 1fr))
/* Result: 7 columns (3 filled, 4 empty ~256px) */

My rule: Use auto-fit for responsive card grids (99% of cases). Use auto-fill when I need to preserve column structure.

Intrinsic Grid Values

Prof. Teeters taught me the grid values that make this magic work:

fr Units - Fractional Distribution

grid-template-columns: 1fr 2fr 1fr;
/* Divide leftover space: 1 part, 2 parts, 1 part */

Why fr > percentages: fr accounts for gaps and content. Percentages don't!

Example: 3 columns with gap: 2rem

  • With %: 33.33% 33.33% 33.33% doesn't account for gaps ‚Üí overflow!
  • With fr: 1fr 1fr 1fr automatically subtracts gaps ‚Üí perfect fit!

minmax(min, max) - Flexible Boundaries

grid-template-columns: minmax(200px, 1fr);
/* Never smaller than 200px, grows to fill */

grid-template-columns: minmax(0, 1fr);
/* Prevents content overflow forcing expansion */

The (0, 1fr) gotcha: minmax(0, 1fr) allows shrinking below content size. Use when long content (URLs, code) would overflow!

Implicit Tracks - Auto Rows

grid-auto-rows: minmax(200px, auto);
/* New rows: minimum 200px, grow to content */

Why this matters: I define columns explicitly, but rows generate automatically. This sets sizing for auto-generated rows!

Custom Properties Pattern

:root {
  --grid-item-min: 16rem;
  --grid-item-max: 1fr;
  --grid-gap: var(--space-md);
}

.grid {
  display: grid;
  grid-template-columns: repeat(
    auto-fit,
    minmax(var(--grid-item-min), var(--grid-item-max))
  );
  gap: var(--grid-gap);
}

Why custom properties: Systematic, maintainable, semantic. Change --grid-item-min once, all grids update!

Bento-Box Layouts

After mastering the magic pattern, Prof. Teeters showed me something amazing: asymmetric "bento-box" layouts where items span multiple tracks, like Japanese bento boxes with different compartment sizes!

Basic spanning pattern:
.grid {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(16rem, 1fr));
  gap: var(--space-md);
}

.featured {
  grid-column: span 2;  /* Takes 2 columns */
  grid-row: span 2;     /* Takes 2 rows */
}

.regular {
  /* Takes 1 column √ó 1 row (default) */
}

Why It Still Works

The grid structure adapts via minmax(), items span intelligently, everything flows based on rules not fixed positions!

Example:

  • Desktop (1440px): 5 columns ‚Üí featured item spans 2 of 5
  • Tablet (768px): 2 columns ‚Üí featured item spans both
  • Mobile (375px): 1 column ‚Üí featured item same as regular (can't span more than exists)

Magazine-Style Layouts

.magazine-grid {
  display: grid;
  grid-template-columns: repeat(4, minmax(16rem, 1fr));
  gap: var(--space-md);
}

.hero {
  grid-column: span 2;
  grid-row: span 2;
}

.sidebar {
  grid-column: span 1;
  grid-row: span 2;
}

.article {
  grid-column: span 2;
}

Creates complex, asymmetric layouts that still adapt fluidly!
