The Three Eras of Web Design

Prof. Teeters showed me that web design has evolved through three major eras. Each one taught us something important about flexibility and adaptation.

Era 1: Tables (1996-2004)

Layout approach: Using <table> elements for page layout

Flexibility: Zero. Everything was fixed width.

Problem: Accessibility nightmare. Screen readers couldn't distinguish layout tables from data tables. Any width change required restructuring the entire table.

Example: <table><tr><td width="300">Content</td></tr></table>

Era 2: Floats + Breakpoints (2005-2016)

Layout approach: CSS floats + media queries at fixed breakpoints

Flexibility: Discrete jumps at 3-4 breakpoints (320px, 768px, 1024px)

Problem: Designer predicts screen sizes. Always playing catch-up with new devices. Layouts jump abruptly, don't adapt smoothly.

Example: float: left; width: 33.33%; with @media (min-width: 768px)

Era 3: Intrinsic Design (2017-present)

Layout approach: Modern CSS: Grid, Flexbox, container queries, intrinsic sizing

Flexibility: Continuous adaptation. Content + available space = design.

Advantage: Browser calculates perfect layout for ANY size. Works everywhere automatically‚Äîno device-specific code.

Example: grid-template-columns: repeat(auto-fit, minmax(16rem, 1fr));

Prof. Teeters showed me the fundamental difference between old and new CSS thinking with a side-by-side comparison.

‚ùå EXTRINSIC: Fixed values, rigid breakpoints

  width: 300px;           /* Force exact size */
  padding: 24px;          /* Fixed spacing */
  background: #f5f5f5;    /* Hex color */
}

@media (min-width: 768px) {
  .card { width: 600px; } /* Predict screen sizes */
}

/* Problems: Breaks at 400px. Wastes space at 1920px.
   Requires constant maintenance for new devices. */

‚úÖ INTRINSIC: Flexible boundaries, adaptive spacing

/* Designing FOR unpredictability with flexible rules */

  width: clamp(16rem, 10rem + 40vw, 60rem);  /* Computed slope: base + scaling */

Notice how the intrinsic version uses clamp() with computed slope formulas? Both width (10rem + 40vw) and padding (0.5rem + 1vw) use base + slope √ó unit, not arbitrary percentages. The middle value describes HOW the size scales across viewport sizes.

Jen Simmons' Three Intrinsic Principles

Prof. Teeters introduced me to Jen Simmons' intrinsic web design principles. These aren't CSS specs‚Äîthey're conceptual rules of thumb that guide how we think about modern CSS.

Principle 1: Contracting & Expanding

What it means: Design adapts fluidly to space changes. Elements grow and shrink within defined boundaries.

Why intrinsic: Element responds to available space, not fixed dimensions.

Example:

width: clamp(16rem, 10rem + 40vw, 60rem);

The element contracts down to 16rem minimum, expands up to 60rem maximum, and scales between them using a computed slope formula (10rem base + 40vw scaling).

Principle 2: Flexibility

What it means: Modern units enable variable-rate adaptation. Tools like fr, minmax(), clamp(), and auto-fit let browsers calculate optimal layouts.

Why intrinsic: Scaling is fluid and continuous, not discrete jumps.

Example:

font-size: clamp(1rem, 0.5rem + 1.5vw, 2rem);

Typography scales smoothly across viewport sizes using a computed slope formula (0.5rem base + 1.5vw scaling).

Principle 3: Viewport-side Usage

What it means: Using all four viewport sides, not just width. Viewport-relative units (vw, vh, vmax, vmin, dvh) enable designs that respond to actual available space.

Why intrinsic: Layouts adapt to the full viewport context.

Example:

min-height: 100dvh; /* Dynamic viewport height */

Note: In Station 5, we'll extend this to container units (cqi, cqb, cqmin) for component-level intrinsic design.

Important Note:

These are conceptual rules of thumb, not CSS specs. They guide thinking about intrinsic design, but they're not standardized terms in the CSS specification. Think of them as design philosophy, not technical requirements.

CSS Custom Properties: The Foundation

Prof. Teeters explained that CSS custom properties aren't just about storing values‚Äîthey're essential for systematic intrinsic design.

Why Custom Properties for Intrinsic Design

1. Semantic naming: --space-md (intent) vs 24px (appearance)

2. Single source of truth: Change once in :root, update everywhere

3. Systematic scales: Thoughtful values, not random numbers

4. Values can BE intrinsic: In Station 3, we'll make custom properties fluid with clamp()

Basic Scale (Station 1 Version)

Start with fixed rem values to establish a spacing system:

:root {
  --space-xs: 0.5rem;
  --space-sm: 1rem;
  --space-md: 1.5rem;
  --space-lg: 2rem;
  --space-xl: 3rem;

  --width-readable: 60ch;
}

.card {
  padding: var(--space-md);
  max-width: var(--width-readable);
}
Evolution to Fluid Scale (Station 3 Preview):
:root {
  --space-md: clamp(1.5rem, 1rem + 1.25vw, 2.5rem);
  /* Computed slope: base + scaling = grows from 1.5rem to 2.5rem */
}

/* Components using var(--space-md) automatically become fluid! */

This is a preview of Station 3. We start simple (fixed values), then upgrade to fluid (computed slopes). Your components using var(--space-md) don't need to change‚Äîthey inherit the fluidity!

Try It Yourself: Convert to Intrinsic Design

Ready to practice? Here's some extrinsic code to convert to intrinsic design using CSS custom properties and flexible boundaries.

Starter Code (Extrinsic - Old Way):
<div class="card">
  <h2>Product Title</h2>
  <p>Description text goes here</p>
  <button>Learn More</button>
</div>

<style>
.card {
  width: 300px;
  padding: 24px;
  margin: 16px;
  background: #f5f5f5;
  border: 1px solid #ddd;
}

button {
  padding: 8px 16px;
  background: #3498db;
  color: white;
}
</style>

Your Tasks:

  1. Create a :root selector with a spacing scale (--space-xs through --space-xl)
  2. Create width constraints (--card-width-min: 16rem, --card-width-max: 40rem)
  3. Create colors using hsl() format (--color-bg, --color-border, --color-primary, --color-text-on-primary)
  4. Replace width: 300px with clamp(var(--card-width-min), 8rem + 40vw, var(--card-width-max))
  5. Replace all padding/margin values with spacing scale CSS custom properties
  6. Replace hex colors with CSS custom properties
Solution (Intrinsic - Modern Way):
:root {
  /* Spacing scale */
  --space-xs: 0.5rem;
  --space-sm: 1rem;
  --space-md: 1.5rem;

  /* Width constraints */
  --card-width-min: 16rem;
  --card-width-max: 40rem;

  /* Colors (hsl format) */
  --color-bg: hsl(0, 0%, 96%);
  --color-border: hsl(0, 0%, 87%);
  --color-primary: hsl(210, 76%, 54%);
  --color-text-on-primary: hsl(0, 0%, 100%);
}

.card {
  /* Intrinsic width with flexible boundaries using computed slope */
  width: clamp(var(--card-width-min), 8rem + 40vw, var(--card-width-max));

  /* Systematic spacing */
  padding: var(--space-md);
  margin: var(--space-sm);

  /* CSS custom properties for colors */
  background: var(--color-bg);
  border: 1px solid var(--color-border);
}

button {
  /* Systematic spacing */
  padding: var(--space-xs) var(--space-sm);

  /* CSS custom properties for colors */
  background: var(--color-primary);
  color: var(--color-text-on-primary);
}

What changed: Fixed pixels ‚Üí systematic custom properties. Hex colors ‚Üí hsl() format. Exact width ‚Üí flexible clamp() boundaries. Random values ‚Üí intentional scale.
