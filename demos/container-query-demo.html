üî¨ How It Works - The Code

Here's the CSS that creates the difference you see above

Bonus: This demo page's layout also uses container queries instead of media queries - even the grid breakpoints at 900px use @container not @media!

‚ùå Viewport Media Query (Extrinsic)

Asks: "How wide is the VIEWPORT?" - Doesn't care about container size!

/* Viewport Media Query */
@media (min-width: 768px) {
  .viewport-card {
    flex-direction: row !important;
    gap: 2rem !important;
    padding: 2rem !important;
  }

  .viewport-card img {
    width: 40% !important;
    flex-shrink: 0;
  }
}

/* Result: Goes horizontal whenever
   VIEWPORT ‚â• 768px, even if the
   container is only 300px wide! */

‚úÖ Container Query (Intrinsic)

Asks: "How wide is MY CONTAINER?" - Adapts to actual available space!

/* Container Query Setup */
.container-card-wrapper {
  container-type: inline-size;
  container-name: demo-card;
}

/* Container Query */
@container demo-card (min-width: 400px) {
  .container-card {
    flex-direction: row !important;
    gap: 2rem !important;
    padding: 2rem !important;
  }

  .container-card img {
    width: 40% !important;
    flex-shrink: 0;
  }
}

/* Result: Only goes horizontal when
   CONTAINER ‚â• 400px, regardless of
   viewport width! */

Container queries are INTRINSIC - they ask 'How wide am I?' and adapt accordingly. That's why my card worked perfectly in the sidebar once I switched to container queries!"
